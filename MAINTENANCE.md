## ?? Issue とプルリクエストのポリシー

### Issue の分類

新しい Issue は以下のラベルで分類されます：

| ラベル | 対応 | 説明 |
|--------|------|------|
| `bug-critical` | ? 対応 | 重大なバグ |
| `bug-minor` | ? 対応 | 軽微なバグ |
| `enhancement` | ? 対応しない | 新機能の要望 |
| `wontfix` | ? 対応しない | .NET 10版で対応予定 |
| `documentation` | ? 対応 | ドキュメントの修正 |

### プルリクエストの受け入れ基準

以下の条件を満たすプルリクエストのみ受け入れます：

? **受け入れ可能:**
- 重大なバグの修正
- セキュリティの脆弱性の修正
- ドキュメントの修正
- テストの修正
- ビルドスクリプトの修正

? **受け入れ不可:**
- 新機能の追加
- 大規模なリファクタリング
- パフォーマンスの最適化（重大な問題を除く）
- .NET Framework 以外への対応

## ?? コントリビューター向けガイド

### バグ報告の方法

1. **既存の Issue を確認**
   - 同じ問題が既に報告されていないか確認

2. **再現手順を明記**
   - OS とバージョン
   - .NET Framework バージョン
   - 実行したコマンド
   - 期待される動作と実際の動作

3. **優先度を判断**
   - Critical: セキュリティ、データ損失、クラッシュ
   - Minor: その他のバグ

### プルリクエストの方法

1. **Issue を作成**して、修正内容を事前に確認

2. **フォークとブランチ作成**
```bash
git checkout -b fix/issue-123
```

3. **最小限の修正**
   - 問題を解決する最小限の変更のみ
   - 既存のテストが全て合格すること
   - 必要に応じて新しいテストを追加

4. **プルリクエスト作成**
   - Issue 番号を参照
   - 変更内容の説明
   - テスト結果のスクリーンショット
